# 先看这里

这个目录是论文仿真代码的清洁室主线版。第一遍阅读时，不要从 `data/` 或 sibling 归档目录开始翻，容易被历史结果和中间文件干扰。

## 5 分钟读懂

按这个顺序看：

1. 根目录 `README.md`：知道目录各自负责什么。
2. `docs/01_仿真主线流程.md`：知道从 Stage1 到论文图的完整流水线。
3. `docs/02_论文图表数据对应关系.md`：知道每张论文图对应哪个 Excel、哪个 export 脚本、哪个 run 脚本。
4. `data/exports/导出Excel索引.md`：真正画图或找 Origin 数据时从这里进。
5. `docs/03_导出Excel使用指南.md`：不确定某个 Excel 是否该用时再看。

## 最重要的主线

论文主线可以压缩成：

```text
run/Stage1Code_CQR.m
run/stage1_cqr.py
run/run_regd_15min_historical_variance_v8.m
run/run_ch3_hourly_reserve_costcurve_main_V2.m
run/run_ch3_baseline_fixedTs_suite_V4.m
run/run_ch3_hourly_credible_feasibility_screen_V2.m
run/run_postprocess_hourly_credible_*_view_full.m
export/export_*.m
data/exports/<按图号分组>/*.xlsx
origin/figure1.opju
paper/v3_TSTE.tex
```

如果只是想看论文图数据，不需要跑主仿真，直接看 `data/exports/导出Excel索引.md`。

## 哪些目录先不要看

- `../99-paper-sim-cleanroom_archive/`: 历史分支、旧图、旧表和引用副本，默认不看。
- `../99-paper-sim-cleanroom_supplement/`: 多测试日稳健性补充实验，默认不看。
- `docs/90_*.md` 和 `docs/91_*.csv` 以后：验证记录和机器清单，不是第一阅读入口。

## 给新接手同学的建议

先回答三个问题：

1. 论文每张图的数据在哪里？看 `docs/02_论文图表数据对应关系.md`。
2. 如果要重新导出 Excel，跑哪个脚本？看 `docs/04_导出脚本说明.md`。
3. 如果要从头复现实验，按什么顺序跑？看 `docs/01_仿真主线流程.md`。
