# 本校学位论文写作风格指南

本文档是本项目的长期写作风格指南，主要用于后续章节写作、语言润色、结构审阅和全文自查。它不直接生成论文正文，也不替代学校正式格式要求、论文最终章节结构或作者的学术判断。

后续写作时，Web GPT 可基于本文档审阅章节逻辑和语言风格；Codex 可基于本文档执行局部润色、术语统一和格式整理，但不得依据本文档自行新增研究内容、实验结论或技术观点。

------

## 1. 适用范围与使用方式

本指南适用于中文本科毕业论文《商业建筑HVAC系统动态需求响应》的后续写作、润色、审阅和自查。论文专业方向属于电气工程与电力系统，写作目标是满足本科毕业论文规范，同时在章节逻辑、学术表达和结果分析深度上适度向硕士论文看齐。

本文档的使用方式如下：

- 用于统一论文的章节功能、语言风格和论证方式；
- 用于检查摘要、绪论、文献综述、模型方法、仿真分析、结论等部分是否符合学位论文写作习惯；
- 用于帮助 Web GPT 和 Codex 在后续协作中遵守同一套写作规则；
- 用于避免正式写作时直接模仿样本论文的研究内容、数据、结论和原创观点。

本文档不用于：

- 直接生成论文正文；
- 直接确定最终目录；
- 代替学校写作指南、导师意见和最终 Word/LaTeX 模板；
- 复用样本论文中的具体研究对象、实验结论、数据结果和原创观点。

后续写作优先级为：

```text
学校本科毕业论文写作指南 > 本文实际研究内容与仿真结果 > 本指南中的风格建议 > 样本论文中的一般写作习惯
```

## 2. 学校写作要求中与本文写作直接相关的规则

完整格式要求以学校写作指南、书写范例和最终论文模板为准。本节只提炼后续写作、润色和审阅中反复用到的规则，不展开字体、字号、页眉、装订等排版细节。

### 2.1 摘要与关键词

- 摘要应具有独立性和自含性，即不阅读全文也能了解论文的必要信息。
- 摘要应包括研究目的、理论与实际意义、主要研究内容、研究方法，并突出研究成果和结论。
- 摘要不能写成章节目录式介绍。
- 中文摘要一般控制在约300字左右，英文摘要应与中文摘要内容一致。
- 摘要中不宜出现公式、图表、非常用缩写和参考文献编号。
- 关键词应为3—6个，体现研究对象、研究问题、研究方法和应用场景。

### 2.2 正文结构

- 正文应包括绪论、论文主体和结论等部分。
- 论文主体应结构严谨、层次清晰、重点突出、文字简练通顺。
- 各章之间应前后关联，构成有机整体。
- 正文不应写成简单综述或工程介绍，而应体现“问题提出—模型或方法—仿真验证—结果分析—结论归纳”的研究闭环。

### 2.3 绪论

- 绪论一般作为第1章。
- 绪论应说明课题来源、研究背景、理论意义与实际意义、国内外研究现状、存在不足、本文研究内容和研究方法。
- 绪论中的文献综述应服务于本文研究问题，不能孤立堆砌。
- “本文主要研究内容”不能简单复制目录，应说明各项研究任务之间的逻辑关系。

### 2.4 论文主体

- 论文主体中的模型、方法、仿真、结果分析应层层衔接。
- 科学实验、计算机仿真的条件、过程和结果分析应交代清楚。
- 不应直接给出结果曲线和结论，而缺少仿真设置、对比对象和原因解释。
- 引用他人成果时应明确标注，不能与本文工作混写。

### 2.5 结论

- 结论应单独排写，不应与摘要混同。
- 结论应总结本文研究过程中完成的工作、得到的主要认识、不足和展望。
- 结论应与正文内容对应，不应提出正文未支撑的新观点。
- 本科论文贡献表达应适度，避免夸大研究价值。

### 2.6 参考文献

- 参考文献应支撑正文中的关键观点、研究现状和方法来源。
- 文献引用应采用顺序编码制。
- 应尽量引用原始文献。
- 参考文献一般不少于15篇，其中应包含一定数量的外文文献和近年文献。
- 文献综述中不能只列文献，应有分类、比较和归纳。

### 2.7 图表

- 图表应具有自明性，并与正文紧密配合。
- 正文中应先引出图表，再展示图表，随后分析图表。
- 图表分析不能只描述“升高、降低、变化”，还应解释产生变化的原因。
- 每张图表都应服务于某个论证目的。

### 2.8 公式

- 公式前应说明建模对象、变量关系或推导目的。
- 公式后应解释符号含义和物理意义。
- 首次出现的变量和参数应说明含义、单位或取值来源。
- 公式不能孤立出现，必须与后续仿真或分析形成联系。

### 2.9 名词术语

- 科技名词和专业术语应规范、统一。
- 英文缩写首次出现时应给出中文名称和英文全称。
- 同一概念全文只使用一种主要译名。
- 变量符号、单位和物理意义应保持一致。

### 2.10 仿真条件与结果分析

- 仿真平台、模型来源、时间尺度、输入数据、关键参数、对比场景和评价指标应交代清楚。
- 仿真结果应包括现象描述、对比分析、原因解释和结论归纳。
- 不应只写“验证了有效性”，而要说明有效性体现在哪些指标或现象上。

### 2.11 本章小结

- 论文主体各章后宜设置“本章小结”。
- 本章小结应概括本章方法、结果和结论。
- 本章小结不应引入新的模型、文献或结论。
- 本章小结应为下一章或最终结论提供支撑。

## 3. 样本论文整体结构规律

本节主要依据本校硕士、博士论文样本提炼写作规律；非本校论文仅作为电气工程类学位论文表达方式的辅助参考，不作为本校格式或风格判断依据。

样本论文仅用于分析章节安排、标题层级、语言风格和论证方式，不用于复用其具体研究内容、实验结论、数据结果和原创观点。

需要强调的是，本节总结的是样本论文中呈现出的常见写作规律和结构习惯，并不属于学校硬性格式要求。后续正式论文结构应以学校写作指南、导师意见、本文实际研究内容和图表证据链为准。

### 3.1 常见结构规律

从已查看样本可以观察到，电力系统方向硕士论文常采用“绪论—对象建模—方法构建—算例验证—结论”的组织方式；电机与控制方向博士论文往往将理论推导、控制策略和实验验证拆分得更细。

本文后续可借鉴其逻辑递进方式，但不宜直接套用博士论文的复杂章节展开。

常见组织顺序为：

```text
研究背景与问题提出
→ 理论基础或系统建模
→ 方法设计或策略构建
→ 仿真/实验验证
→ 结果分析与总结
```

### 3.2 标题层级规律

样本论文通常使用三级标题：

```text
第1章
1.1
1.1.1
```

本科论文可采用三级标题作为主要层级，不宜过多使用四级标题。层级过多会削弱论文主线，使正文显得零散。

### 3.3 各类章节功能

| 章节类型       | 常见功能                                 | 本科论文适配建议       |
| -------------- | ---------------------------------------- | ---------------------- |
| 绪论           | 说明背景、意义、研究现状、问题和本文工作 | 适合借鉴               |
| 理论基础章     | 解释核心概念、系统结构、基本原理         | 适合，但不宜写成教材   |
| 建模章         | 建立对象模型、变量关系和约束条件         | 适合，应控制复杂度     |
| 方法章         | 提出控制、优化、评估或响应策略           | 适合，但应围绕单一主线 |
| 仿真章         | 设置场景、展示结果、分析机制             | 非常适合               |
| 扩展应用章     | 多场景、多目标、多约束扩展               | 本科论文谨慎使用       |
| 深层理论推导章 | 大量数学证明或复杂算法分析               | 不建议直接套用         |

### 3.4 本科论文适配原则

本科毕业论文可借鉴硕博论文的问题递进、章节衔接和结果分析方式，但应控制理论推导和方法展开深度，避免形成过度复杂的多问题、多模型、多场景结构。章节安排应服务于一个清晰主线，而不是追求覆盖过多研究点。

## 4. 摘要与关键词写作规则

本科论文摘要可以借鉴硕士论文“研究背景—问题提出—方法说明—主要结果—结论意义”的信息顺序，但不宜写成过长的多创新点摘要。摘要应突出本文的主要工作和主要结论，避免堆叠过多技术细节、公式、图表编号或背景铺垫。

### 4.1 中文摘要框架

推荐信息顺序：

```text
研究背景：说明研究问题产生的现实背景。
问题提出：指出已有方法或实际应用中存在的不足。
研究目标：说明本文围绕什么问题展开。
研究方法：概括使用的模型、方法、仿真或分析手段。
主要工作：分层说明本文完成的几个方面。
结果结论：概括仿真或分析得到的主要认识。
研究意义：说明本文工作的应用价值或参考意义。
```

### 4.2 英文摘要规则

- 英文摘要应与中文摘要内容一致。
- 不应新增中文摘要没有的信息。
- 技术名词应保持前后一致。
- 句式应清晰，不宜堆叠过长从句。
- 结果和结论应明确，不应只描述研究背景。

### 4.3 摘要应避免

摘要应避免：

- 写成章节目录式介绍；
- 只介绍研究背景而缺少方法和结果；
- 夸大研究贡献；
- 出现未在正文中充分支撑的结论；
- 使用过多非常用缩写和未解释术语；
- 直接标注参考文献编号。

### 4.4 关键词规则

关键词应覆盖研究对象、研究问题、研究方法、应用场景和关键约束或指标。关键词不宜过泛，也不宜全部是缩写。

## 5. 绪论写作规则

绪论不应只是背景材料和文献的堆砌，而应形成“现实背景—工程需求—学术问题—已有研究不足—本文切入点”的递进链条。每一小节都应服务于引出本文研究问题，而不是孤立介绍领域知识。

### 5.1 推荐逻辑链条

```text
宏观背景
→ 工程需求
→ 研究对象
→ 学术问题
→ 已有研究不足
→ 本文切入点
→ 本文研究内容与组织安排
```

### 5.2 绪论各部分功能

| 部分           | 作用                 | 写作重点                         |
| -------------- | -------------------- | -------------------------------- |
| 研究背景       | 说明问题为什么重要   | 从电力系统运行需求或能源转型切入 |
| 研究意义       | 说明研究有什么价值   | 区分工程意义和理论意义           |
| 国内外研究现状 | 说明已有研究做了什么 | 按主题分类，而不是逐篇罗列       |
| 存在问题       | 说明仍有什么不足     | 必须与本文研究内容对应           |
| 本文研究内容   | 说明本文做什么       | 不能直接复制目录                 |
| 技术路线       | 说明如何完成研究     | 可用文字或流程图表达             |
| 章节安排       | 说明全文组织         | 简洁说明每章功能                 |

### 5.3 面向本项目的绪论启发

对于《商业建筑HVAC系统动态需求响应》方向，绪论中不宜直接从“空调负荷很大”跳到“本文提出方法”，而应逐步说明：

新型电力系统需要灵活调节资源，商业建筑 HVAC 负荷具备可调节潜力，但其参与动态需求响应时受到热舒适、设备运行状态、水-空气耦合特性和可信可调能力评估等因素约束，因此需要进一步研究其可调能力表征和运行决策方法。

以上内容仅为写作启发，不是最终正文。

## 6. 文献综述写作规则

文献综述应优先围绕问题分类，而不是按作者或年份机械罗列。每一类文献综述后应有简短归纳，说明该类方法解决了什么问题、仍存在什么不足，以及这些不足如何引出本文后续工作。

### 6.1 分类综述

可采用的分类方式包括：

```text
按研究对象分类
按方法类型分类
按应用场景分类
按控制目标分类
按不确定性处理方式分类
按仿真或实验验证方式分类
```

对于本项目，后续可暂定从以下方向组织综述。这些分类只是当前阶段的写作准备方向，不是最终文献综述框架；正式分类仍需结合实际检索到的文献、本文研究问题和最终章节安排进一步调整：

```text
需求响应机制研究
建筑柔性负荷建模研究
HVAC系统控制策略研究
动态响应与仿真验证研究
备用能力或可调能力评估研究
```

### 6.2 推荐与不推荐写法

不推荐：

```text
已有研究[1]提出……；研究[2]提出……；研究[3]提出……。
```

推荐：

```text
现有研究主要从负荷建模、控制策略和灵活性评估三个方面展开。负荷建模类方法能够刻画建筑热动态过程，但在面向备用服务时对可交付能力的刻画不足；控制策略类方法关注运行优化，但对市场报价或备用供给函数的支撑仍有限。
```

### 6.3 引出本文问题

每一类文献综述后应回答：

```text
这类研究解决了什么？
仍存在什么不足？
该不足与本文有什么关系？
```

综述最后应自然转入本文工作，但应避免使用“填补空白”“首次提出”等强表述，除非后续有充分证据。

## 7. 理论基础、模型方法与仿真分析章节写作规则

### 7.1 理论基础写法

理论基础章应服务于后续建模和仿真，不应写成教材。

推荐顺序：

```text
说明为什么需要该理论
→ 介绍与本文直接相关的核心概念
→ 说明该理论如何用于本文模型或方法
→ 指出本章内容与下一步分析的关系
```

### 7.2 模型与方法写法

模型与方法章节应避免只堆公式。每组公式前应先说明建模对象、变量含义、基本假设和该模型在论文中的作用；公式后应解释该表达式如何支撑后续仿真或分析。

推荐顺序：

```text
研究对象说明
→ 系统边界
→ 基本假设
→ 变量定义
→ 模型建立
→ 参数说明
→ 约束条件
→ 模型用途
```

### 7.3 仿真环境与参数设置

仿真章节应交代：

```text
仿真平台
模型来源
时间尺度
输入数据
关键参数
对比场景
评价指标
```

如果使用已有仿真模型、实测数据或小论文材料，应说明来源、处理方式和适用范围。

### 7.4 仿真结果分析规则

仿真分析不应停留在“曲线升高/降低”“结果更好”这类现象描述，而应进一步解释原因。结果分析建议遵循：

1. 说明仿真场景和对比对象；
2. 引出图表；
3. 描述关键现象；
4. 比较不同方法、参数或场景的差异；
5. 从模型机制、物理机制或控制逻辑角度解释原因；
6. 回到本节研究问题，归纳可支撑的结论。

### 7.5 面向 HVAC 动态需求响应的结果分析要求

对于 HVAC 动态需求响应相关结果，图表分析应尽量从“负荷变化现象”推进到“水-空气耦合、热舒适约束、设备运行点、动态响应能力、可调能力可信性及成本变化机制”等方面的解释，避免只描述功率曲线变化。

每张图表后应尽量回答：

```text
图中哪个变量发生变化？
变化方向和幅度如何？
这种变化由什么机制导致？
是否满足响应目标或约束条件？
对动态需求响应能力说明了什么？
```

## 8. 结论与展望写作规则

结论不是摘要的重复。摘要面向全文概览，结论面向研究完成后的归纳，应更强调本文实际完成的工作、得到的主要认识、存在不足和后续展望。

### 8.1 常见结构

```text
全文工作概括
→ 主要研究结论
→ 本文不足
→ 后续展望
```

### 8.2 主要结论写法

每条结论应包含：

```text
完成的工作
→ 得到的结果或认识
→ 说明的意义或作用
```

结论应与正文各章对应，不应提出正文没有分析过的新内容。

### 8.3 本科论文贡献表达边界

本科毕业论文可以表达“完成了某项建模、仿真、分析或验证工作”，但应谨慎使用“首次”“显著突破”“完全解决”等强贡献表述。贡献表达应与实际工作量和仿真结果相匹配。

推荐表达：

```text
本文结果可为……提供参考。
本文分析有助于理解……
仿真结果表明，所建模型能够反映……
```

避免表达：

```text
彻底解决了……
具有重大突破……
首次实现了……
完全证明了……
```

## 9. 中文学术语言风格指南

样本论文共同体现出的中文学术表达特征包括：语气客观、句式完整、重视因果关系、避免口语化、较少使用绝对化判断，并常用“针对……问题”“为……提供依据”“结果表明……”等表达方式。

### 9.1 推荐表达方式 / 应避免表达方式 / 适用场景

| 适用场景     | 推荐表达方式                         | 应避免表达方式         |
| ------------ | ------------------------------------ | ---------------------- |
| 引出研究背景 | 随着……的发展，……问题逐渐受到关注     | 现在……很火             |
| 说明研究意义 | 对……具有一定的理论意义和工程参考价值 | 这个研究非常重要       |
| 提出问题     | 然而，现有研究在……方面仍存在不足     | 但是别人都没做好       |
| 引出本文工作 | 基于上述分析，本文围绕……开展研究     | 所以我要研究……         |
| 说明方法     | 本文采用……方法，对……进行建模与分析   | 我用了……方法           |
| 说明仿真     | 为验证……，设置……仿真场景             | 为了看看效果，做了仿真 |
| 描述结果     | 仿真结果表明，……                     | 从图里可以看出挺好     |
| 分析原因     | 产生上述现象的原因在于……             | 这是因为它比较好       |
| 总结章节     | 本章围绕……展开，建立了……             | 这一章讲完了           |
| 表达不足     | 受限于……，本文尚未进一步考虑……       | 本文还有很多问题       |
| 表达贡献     | 可为……提供参考                       | 具有巨大应用价值       |

### 9.2 客观化表达

推荐：

```text
本文围绕……开展研究。
结果表明，……
可以看出，……
进一步说明，……
```

避免：

```text
我认为……
我们可以轻松发现……
效果非常好……
这个方法很厉害……
```

### 9.3 克制贡献表达

推荐：

```text
在一定程度上提高了……
为……提供了参考。
可为……提供支撑。
有助于改善……
```

避免：

```text
彻底解决了……
具有颠覆性意义……
达到了世界领先水平……
完全证明了……
```

### 9.4 图表分析表达

推荐：

```text
由图X可知，……
与……相比，……呈现出……
造成这一现象的原因在于……
这说明……
```

避免：

```text
图X很明显。
结果非常好。
曲线变化很大。
```

### 9.5 过渡与总结表达

段落或章节之间应使用过渡句增强逻辑衔接。

推荐：

```text
在明确……的基础上，下一步需要……
上述分析为……提供了基础。
然而，仅考虑……仍难以反映……
因此，有必要进一步……
由此可见，……
上述结果说明，……
```

## 10. 段落组织与论证逻辑规则

学术论文正文段落不应只是材料罗列，而应形成小型论证单元。不同类型段落可采用不同组织模板。

### 10.1 背景引出型段落

```text
背景现象 → 工程问题 → 研究必要性 → 本文切入点
```

适用于绪论中的研究背景、研究意义和问题提出部分。

### 10.2 文献综述型段落

```text
研究类别 → 代表性工作 → 已解决问题 → 仍存在不足 → 与本文关系
```

适用于国内外研究现状部分。重点是归纳类别和引出问题，而不是逐篇复述。

### 10.3 方法说明型段落

```text
研究目标 → 建模对象 → 方法思路 → 关键变量/约束 → 后续用途
```

适用于理论基础、模型方法、控制策略、优化方法等章节。

### 10.4 图表分析型段落

```text
图表引出 → 现象描述 → 对比分析 → 原因解释 → 小结论
```

适用于仿真分析和结果讨论部分。

### 10.5 本章小结型段落

```text
本章工作 → 采用方法 → 得到结果 → 对下一章的支撑
```

适用于每章末尾的“本章小结”。

## 11. 面向《商业建筑HVAC系统动态需求响应》的写作启发

本节仅用于指导后续写作表达，不作为本文最终贡献点、章节结构或技术路线的正式确认。最终内容安排需要结合小论文材料、仿真结果、图表证据链和学校要求进一步确定。

### 11.1 研究背景切入

研究背景应从新型电力系统灵活性需求引出，而不是直接从 HVAC 设备介绍开始。可采用如下递进思路：

```text
新型电力系统运行灵活性需求增加
→ 需求响应成为重要调节手段
→ 商业建筑具有较大可调负荷潜力
→ HVAC系统是商业建筑中重要柔性负荷
→ 动态需求响应需要考虑系统运行过程和舒适性约束
```

### 11.2 HVAC 系统定位

HVAC 系统应作为商业建筑柔性负荷资源展开，而不是只作为普通空调系统描述。写作中应说明其负荷调节与室内热环境、设备运行状态、控制策略和系统动态过程有关。

### 11.3 结果分析写法

仿真结果章节应避免只堆叠图表。每张图都应说明其支撑哪个结论，并尽量从“现象描述”推进到“机制解释”。

图表分析应关注：

```text
负荷变化现象
→ 控制变量变化
→ 室温或舒适约束响应
→ 设备运行点变化
→ 动态响应能力、可调能力可信性或响应成本变化
```

### 11.4 小论文到本科论文的转换

后续写作中应特别注意“小论文到本科毕业论文”的转换：英文小论文强调创新点集中和表达精炼，本科毕业论文则需要更完整地交代背景、建模过程、仿真设置、结果分析和工程意义。因此，不能直接翻译小论文，而应按照中文本科毕业论文的叙事逻辑重新组织。

### 11.5 贡献表达

贡献表达要克制，避免把本科论文写成过度夸张的高水平论文口吻。更适合的定位是：围绕商业建筑 HVAC 系统动态需求响应问题，完成系统建模、仿真场景设计、响应过程分析和结果机理解释，为理解 HVAC 负荷的动态响应能力与可调能力可信性提供参考。

后续写作中可强调本文完成了以下类型的工作：

```text
建立或整理了面向动态需求响应分析的 HVAC 系统模型；
设置了能够反映不同运行条件的仿真场景；
分析了控制变量、负荷功率和室内热环境之间的动态关系；
结合仿真结果解释了 HVAC 系统可调能力变化的原因；
为商业建筑柔性负荷参与需求响应提供了建模与仿真层面的参考。
```

贡献表达应与正文实际完成内容保持一致，不应使用“首次提出”“彻底解决”“显著突破”等缺乏充分支撑的表述。

## 12. 后续写作检查清单

### 12.1 章节结构检查

-  本章功能是否明确？
-  本章是否服务于全文主线？
-  小节顺序是否符合“问题—方法—结果—结论”的递进？
-  章节之间是否存在自然承接？
-  是否存在与主题关系不强的章节？

### 12.2 段落逻辑检查

-  每段是否有明确中心句？
-  段内是否存在只罗列、不归纳的问题？
-  段落之间是否有过渡？
-  段落结尾是否形成必要归纳？
-  是否存在一个段落包含多个互不相关主题的问题？

### 12.3 文献综述检查

-  是否按问题分类，而不是按文献逐篇堆叠？
-  是否比较了不同研究思路？
-  是否指出已有研究不足？
-  是否自然引出本文工作？
-  引用是否支撑正文观点？

### 12.4 图表与结果分析检查

-  图表是否先引出再展示？
-  图表是否有明确分析目的？
-  是否解释了结果变化原因？
-  是否从图表中提炼了可支撑的结论？
-  是否存在“只展示曲线、不分析机制”的问题？

### 12.5 学术语言检查

-  是否存在口语化表达？
-  是否存在夸大贡献？
-  是否存在主观评价？
-  术语是否统一？
-  缩写是否首次解释？
-  句子之间是否有逻辑衔接？

### 12.6 本科论文适配检查

-  理论深度是否适合本科论文？
-  是否过度套用硕博论文复杂结构？
-  是否充分体现完整工作量？
-  是否避免了不必要的过度创新表述？
-  是否体现“本科论文向硕士论文看齐”的规范性，但没有超出实际工作基础？

### 12.7 摘要与结论检查

-  摘要是否包含研究目的、方法、主要工作、结果和结论？
-  摘要是否避免写成目录？
-  英文摘要是否与中文摘要一致？
-  结论是否不是摘要重复？
-  结论是否对应正文工作？
-  是否适度表达贡献，避免夸大？

### 12.8 本章小结检查

-  本章小结是否总结了本章核心工作？
-  是否概括了本章得到的主要认识？
-  是否避免引入新内容？
-  是否为下一章形成承接？
-  是否避免写成简单流水账？
