# 本校学位论文写作风格指南

本文档是本项目的长期写作风格指南，主要用于后续章节写作、语言润色、结构审阅和全文自查。它不直接生成论文正文，也不替代学校正式格式要求、论文最终章节结构或作者的学术判断。

后续写作时，Web GPT 可基于本文档审阅章节逻辑和语言风格；Codex 可基于本文档执行局部润色、术语统一和格式整理，但不得依据本文档自行新增研究内容、实验结论或技术观点。

---

## 1. 适用范围与使用方式

本指南适用于中文本科毕业论文《商业建筑HVAC系统动态需求响应》的后续写作、润色、审阅和自查。论文专业方向属于电气工程与电力系统，写作目标是满足本科毕业论文规范，同时在章节逻辑、学术表达和结果分析深度上适度向硕士论文看齐。

本文档的使用方式如下：

* 用于统一论文的章节功能、语言风格和论证方式；
* 用于检查摘要、绪论、文献综述、模型方法、仿真分析、结论等部分是否符合学位论文写作习惯；
* 用于帮助 Web GPT 和 Codex 在后续协作中遵守同一套写作规则；
* 用于避免正式写作时直接模仿样本论文的研究内容、数据、结论和原创观点。

本文档不用于：

* 直接生成论文正文；
* 直接确定最终目录；
* 代替学校写作指南、导师意见和最终 Word/LaTeX 模板；
* 复用样本论文中的具体研究对象、实验结论、数据结果和原创观点。

后续写作优先级为：

```text
学校本科毕业论文写作指南 > 导师要求 > 本文实际研究内容与仿真结果 > 本指南中的风格建议 > 样本论文中的一般写作习惯
```

## 2. 学校本科毕业论文写作硬性要求

本部分根据学校本科毕业论文写作指南提炼，属于后续论文写作、润色和审阅时必须遵守的基础规则。学校指南明确将本科毕业论文定位为具有一定科研价值和实用价值的学术论文，并要求撰写实事求是、符合相关标准和汉语语法规范。

完整格式要求以学校写作指南、书写范例和最终论文模板为准。本节只提炼写作相关规则，不展开字体、字号、页眉、装订等排版细节。

### 2.1 摘要与关键词

学校指南要求摘要应具有独立性和自含性，能够使读者不阅读全文也能了解论文的必要信息；摘要应包括研究目的、理论与实际意义、主要研究内容、研究方法，并重点突出研究成果和结论。

写作规则：

* 摘要应完整、客观、准确，做到不遗漏、不拔高、不添加；
* 摘要不能写成章节目录式介绍；
* 中文摘要一般控制在约300字左右；
* 英文摘要应与中文摘要内容一致；
* 摘要中不宜出现公式、图表、非常用缩写和参考文献编号；
* 关键词应为3—6个，体现研究对象、研究问题、研究方法和应用场景；
* 关键词应尽量采用规范术语，避免随意自造表达。

### 2.2 正文结构

学校指南规定论文正文包括绪论、论文主体和结论等部分。

写作规则：

* 正文应体现完整研究过程；
* 正文不应写成简单综述或工程介绍；
* 正文应形成“问题提出—模型或方法—仿真验证—结果分析—结论归纳”的研究闭环；
* 各章之间应前后关联，构成有机整体；
* 章节安排应服务于论文主题，而不是机械追求章节数量。

### 2.3 绪论

学校指南要求绪论一般作为第1章，应包括课题来源、研究背景、理论意义与实际意义、国内外研究进展、存在不足，以及本文拟开展的研究内容、研究程序和方法。

写作规则：

* 绪论应服务于论文主题；
* 研究背景应逐步引出研究问题；
* 文献综述不能孤立堆砌；
* 研究现状之后必须自然引出“仍需解决的问题”；
* “本文主要研究内容”不能简单复制目录；
* 研究内容宜体现任务之间的逻辑关系；
* 文献引用应准确支撑对应观点。

### 2.4 论文主体

学校指南强调，论文主体应结构严谨、层次清晰、重点突出、文字简练通顺；各章之间应前后关联，构成有机整体；对科学实验、计算机仿真的条件、实验过程和仿真过程等需加以叙述，避免直接给出结果、曲线和结论。

写作规则：

* 每一章应有明确功能；
* 每一节应围绕本章目标展开；
* 建模、方法、仿真和结果分析之间要形成因果链；
* 不能只放图表而缺少解释；
* 不能把他人成果和本文工作混写；
* 数据、推理和结论必须对应；
* 结果分析应体现“现象—原因—结论”的递进。

### 2.5 结论

学校指南要求结论作为正文组成部分单独排写，不加章标题序号，不标注文献引用；结论不能与摘要混同，内容一般为400—2000字。

写作规则：

* 结论不是摘要的重复；
* 结论应对应全文研究内容；
* 结论应突出“本文完成了什么、得到了什么认识”；
* 结论不应提出正文没有支撑的新观点；
* 本科论文应适度表达贡献，避免使用过度夸大的词语；
* 展望应基于本文不足，而不是泛泛而谈。

### 2.6 参考文献

学校指南要求所有被引用文献均应列入参考文献，采用顺序标注；本科毕业论文参考文献一般不少于15篇，其中外文文献一般不少于3篇，近五年文献一般不少于总数的三分之一，并应有近两年的参考文献。

写作规则：

* 引用文献必须进入参考文献列表；
* 同一篇文献只用一个序号；
* 尽量引用原始文献；
* 参考文献应支撑正文中的关键观点、研究现状和方法来源；
* 文献综述中不能只列文献，应有分类、比较和归纳；
* 不能用参考文献替代自己的分析。

### 2.7 图表

学校指南要求插图、插表应与正文紧密配合，图表应具有自明性；正文中应先有文字提示，再给出图表，随后进行分析。

写作规则：

* 正文中必须先引出图表；
* 图表应有明确分析目的；
* 图表后必须解释图表说明了什么；
* 图表分析不能只描述“升高、降低、变化”，还要解释原因；
* 图表应服务于论证，不应作为装饰；
* 同一类图表的命名和变量表达应保持一致。

### 2.8 公式

学校指南对公式编号、引用和书写有规范要求。对后续写作而言，更重要的是公式必须与正文说明和变量解释相结合。

写作规则：

* 公式前应说明为什么需要该公式；
* 公式后应解释符号含义；
* 首次出现的变量必须说明；
* 公式不能孤立出现；
* 推导复杂时，应说明关键假设和物理意义；
* 公式应服务于建模、仿真或分析，不应为显得复杂而堆砌。

### 2.9 名词术语

学校指南要求科技名词术语和设备、元件名称应采用国家标准或行业通用术语，全文名词术语必须统一。

写作规则：

* HVAC、DR、VPP、AHU、CQR 等缩写首次出现时应给出中文名称和英文全称；
* 同一概念全文只使用一种主要译名；
* “空调负荷”“HVAC负荷”“建筑柔性负荷”等概念要区分使用；
* 变量符号、单位和物理含义应保持一致；
* 避免同一术语在不同章节中出现不同翻译。

### 2.10 仿真条件与结果分析

学校指南特别强调，对于科学实验、计算机仿真的条件、实验过程和仿真过程等需加以叙述，避免直接给出结果、曲线和结论。

写作规则：

* 仿真平台、模型来源、时间尺度、输入数据、关键参数、对比场景和评价指标应交代清楚；
* 仿真结果应包括现象描述、对比分析、原因解释和结论归纳；
* 不应只写“验证了有效性”，而要说明有效性体现在哪些指标或现象上；
* 每组结果应说明比较对象；
* 结果分析应回扣研究问题。

### 2.11 本章小结

学校指南指出，论文主体各章后应有“本章小结”，实验方法或材料等章节可不写；本章小结是对本章研究内容、方法与成果的简洁准确概括，也是最后结论的依据。

写作规则：

* 本章小结不应重复章节开头；
* 不应写成流水账；
* 应总结本章解决了什么问题；
* 应点明本章结果如何支撑下一章；
* 不宜引入新的文献、新模型或新结论。

## 3. 样本论文整体结构规律

本节总结的是样本论文中呈现出的常见写作规律和结构习惯，并不属于学校硬性格式要求。后续正式论文结构应以学校写作指南、导师意见、本文实际研究内容和图表证据链为准。

书写范例本身也明确说明，规范中引用的示例只作为本科毕业论文书写格式的示范，并不代表研究内容的示范。 因此，本节只提炼章节组织、标题层级和行文方式，不学习样本论文的具体研究内容。

### 3.1 常见章节组成

从已查看样本可以观察到，电气工程、电力系统、控制与机电类学位论文普遍采用如下结构：

```text
第1章 绪论
第2章 理论基础、系统建模或方法基础
第3章 核心方法、控制策略或评估方法
第4章 优化模型、仿真验证或实验分析
第5章 进一步应用、系统实现或综合验证
结论
参考文献
```

例如，构网型储能相关硕士论文采用“绪论—系统建模—能力评估—优化配置—结论”的推进方式，其目录中第2章为实用化建模，第3章为支撑能力评估，第4章为优化配置研究。 这说明电力系统类学位论文通常强调从对象建模到指标评估，再到仿真或优化分析的递进关系。

本科论文可以借鉴这种“问题—模型—方法—验证”的基本逻辑，但不宜照搬硕博论文的复杂展开。

### 3.2 常见章节顺序

较成熟的学位论文通常遵循：

```text
背景与问题提出
→ 理论基础或系统建模
→ 方法设计或策略构建
→ 仿真/实验验证
→ 结果分析与总结
```

这种顺序适合本科论文借鉴，因为它能体现研究闭环。

### 3.3 标题层级规律

样本论文普遍使用三级标题：

```text
第1章
1.1
1.1.1
```

本科论文也应以三级标题为主，避免过多四级标题。四级标题过多会使论文显得零散，不利于突出主线。

### 3.4 各章功能规律

| 章节类型 | 常见功能 | 本科论文是否适合借鉴 |
| --- | --- | --- |
| 绪论 | 说明背景、意义、研究现状、问题和本文工作 | 适合 |
| 理论基础章 | 解释概念、系统结构、基本原理 | 适合，但不宜写成教材 |
| 建模章 | 建立对象模型、变量关系、约束条件 | 适合 |
| 方法章 | 提出控制、优化、评估或响应策略 | 适合，但复杂度需控制 |
| 仿真章 | 设置场景、展示结果、分析机制 | 非常适合 |
| 扩展应用章 | 多场景、多目标、多约束扩展 | 本科论文谨慎使用 |
| 深层理论推导章 | 大量数学证明和复杂算法收敛性分析 | 不建议直接套用 |
| 多方法并列创新章 | 每章提出一个独立创新点 | 不建议本科论文机械模仿 |

### 3.5 本科论文适配原则

本科毕业论文可借鉴硕博论文的问题递进、章节衔接和结果分析方式，但应控制理论推导和方法展开深度，避免形成过度复杂的多问题、多模型、多场景结构。章节安排应服务于一个清晰主线，而不是追求覆盖过多研究点。

## 4. 摘要与关键词写作规则

摘要不是目录说明，而是全文的压缩表达。样本论文的摘要通常按照“背景—问题—方法—工作—结果—意义”的顺序组织。

构网型储能硕士论文摘要先说明新型电力系统背景和频率支撑问题，再概括建模、评估和优化配置等主要工作，最后归纳优化结果和意义。 这种写法体现了学位论文摘要常见的信息组织方式，但后续写作不能复用其具体研究内容。

### 4.1 中文摘要常见框架

推荐框架：

```text
研究背景：说明研究问题产生的现实背景。
问题提出：指出已有方法或实际应用中存在的不足。
研究目标：说明本文围绕什么问题展开。
研究方法：概括使用的模型、方法、仿真或分析手段。
主要工作：分层说明本文完成的几个方面。
结果结论：概括仿真或分析得到的主要认识。
研究意义：说明本文工作的应用价值或参考意义。
```

本科论文摘要宜控制在“完整但不过度拔高”的程度。

### 4.2 英文摘要写作规则

英文摘要应与中文摘要内容一致，不应新增中文摘要没有的信息。

英文摘要要注意：

* 避免逐字硬译；
* 优先使用清晰的学术表达；
* 技术名词保持一致；
* 不使用过长复合句；
* 结果和结论应明确；
* 不应只翻译背景而弱化方法和结果。

### 4.3 关键词写作规则

关键词应覆盖：

```text
研究对象
研究问题
研究方法
应用场景
关键约束或指标
```

对于本文方向，后续可能涉及的关键词类型包括：

```text
商业建筑
HVAC系统
动态需求响应
柔性负荷
仿真分析
```

此处仅为类型启发，不确定最终关键词。

## 5. 绪论写作规则

绪论的核心任务是把读者从宏观背景逐步带到本文研究问题。成熟学位论文的绪论通常不是简单堆材料，而是层层缩小范围。

学校指南也要求绪论应包括课题来源、背景、意义、研究现状、不足和本文研究内容方法，并提醒不能将论文目录直接作为研究内容。

### 5.1 推荐逻辑推进方式

```text
宏观背景
→ 电力系统或能源转型需求
→ 需求响应的重要性
→ 建筑/HVAC负荷的可调节潜力
→ 当前研究或应用中的不足
→ 本文拟解决的问题
→ 本文研究内容与技术路线
```

### 5.2 绪论各部分功能

| 部分 | 作用 | 写作重点 |
| --- | --- | --- |
| 研究背景 | 说明问题为什么重要 | 从行业或系统需求切入 |
| 研究意义 | 说明研究有什么价值 | 区分理论意义和工程意义 |
| 国内外研究现状 | 说明别人做了什么 | 按主题分类，而不是按作者罗列 |
| 存在问题 | 说明仍有什么不足 | 必须与本文研究内容对应 |
| 本文研究内容 | 说明本文做什么 | 不直接复制目录 |
| 技术路线 | 说明如何完成研究 | 可用文字或流程图 |
| 章节安排 | 说明全文组织 | 简洁说明每章功能 |

### 5.3 避免材料堆砌的方法

绪论中每一类文献综述之后都应回答：

```text
这类研究解决了什么？
仍存在什么不足？
这个不足与本文有什么关系？
```

如果某段文献不能服务于本文问题，就不应放入绪论。

## 6. 文献综述写作规则

文献综述的目的不是展示读过多少文献，而是为本文研究问题建立依据。

### 6.1 分类综述

推荐按照问题类别分类，而不是按时间或作者简单排列。

可使用的分类方式包括：

```text
按研究对象分类
按方法类型分类
按应用场景分类
按控制目标分类
按不确定性处理方式分类
按仿真或实验验证方式分类
```

对于本项目，后续可暂定从以下方向组织综述。这些分类只是当前阶段的写作准备方向，不是最终文献综述框架；正式分类仍需结合实际检索到的文献、本文研究问题和最终章节安排进一步调整：

```text
需求响应机制研究
建筑柔性负荷建模研究
HVAC系统控制策略研究
动态响应与仿真验证研究
备用能力或可调能力评估研究
```

### 6.2 比较不同研究

文献综述中应避免连续堆叠：

```text
某某研究了……
某某提出了……
某某分析了……
```

更推荐使用比较式写法：

```text
一类研究侧重于……
另一类研究则关注……
相比之下，……方法更适用于……
然而，在……方面仍存在不足。
```

### 6.3 指出不足

指出不足时应克制，避免否定性过强。

推荐方向：

* 研究对象简化；
* 动态过程刻画不足；
* 控制变量耦合考虑不足；
* 仿真场景有限；
* 工程应用条件约束不充分；
* 结果分析偏现象描述，机制解释不足。

### 6.4 引出本文工作

综述最后应自然转入：

```text
基于上述研究基础，本文围绕……问题展开研究。
```

但不能夸大为“填补空白”“首次提出”等，除非有充分证据。

## 7. 理论基础、模型方法与仿真分析章节写作规则

### 7.1 理论基础写法

理论基础章应服务于后续建模和仿真，不应写成教材。

推荐写法：

```text
先说明该理论为什么需要；
再介绍与本文直接相关的核心概念；
然后说明该理论如何用于本文模型或方法；
最后指出本章内容与下一章的关系。
```

避免：

* 大段介绍通用概念；
* 与本文无关的理论背景；
* 只写定义，不说明用途；
* 把综述内容和理论基础混在一起。

### 7.2 模型、变量、假设和约束说明

模型章节应按照以下顺序组织：

```text
研究对象说明
→ 系统边界
→ 变量定义
→ 基本假设
→ 模型建立
→ 参数说明
→ 约束条件
→ 模型用途
```

写作时应特别注意：

* 每个变量首次出现要解释含义；
* 每个假设要说明合理性；
* 约束条件要对应实际物理或工程限制；
* 模型不是越复杂越好，应与本科论文深度匹配。

### 7.3 仿真环境、参数设置和数据来源

仿真章节应交代：

```text
仿真平台
模型来源
时间尺度
输入数据
关键参数
对比场景
评价指标
```

如果使用已有仿真模型或数据，应说明来源和处理方式。

### 7.4 图表分析

图表分析应遵循：

```text
引出图表
→ 描述主要现象
→ 比较关键差异
→ 分析产生原因
→ 提炼本节结论
```

不推荐只写：

```text
由图可知，曲线发生变化。
```

应进一步说明：

```text
变化反映了什么机制？
与控制策略或模型假设有什么关系？
是否支持本文观点？
```

### 7.5 结果分析

结果分析应至少包含三层：

| 层次 | 内容 |
| --- | --- |
| 现象描述 | 图表中出现了什么变化 |
| 原因解释 | 为什么会出现这种变化 |
| 结论归纳 | 这种变化说明了什么 |

### 7.6 避免“只展示结果，不解释原因”

每个仿真结果后都应检查：

```text
是否说明了对比对象？
是否说明了指标变化？
是否解释了变化原因？
是否回扣了研究问题？
是否形成了明确结论？
```

### 7.7 面向 HVAC 动态需求响应的结果分析要求

对于 HVAC 动态需求响应相关结果，图表分析应尽量从“负荷变化现象”推进到“水-空气耦合、热舒适约束、设备运行点、动态响应能力、可调能力可信性及成本变化机制”等方面的解释，避免只描述功率曲线变化。

每张图表后应尽量回答：

```text
图中哪个变量发生变化？
变化方向和幅度如何？
这种变化由什么机制导致？
是否满足响应目标或约束条件？
对动态响应能力、可调能力可信性或响应成本说明了什么？
```

## 8. 结论与展望写作规则

### 8.1 结论常见结构

结论通常包括：

```text
全文工作概括
→ 主要研究结论
→ 本文不足
→ 后续展望
```

本科论文中，结论应控制在稳妥、客观、可验证的范围内。

### 8.2 研究工作总结

开头应概括本文围绕什么问题，开展了哪些工作。

避免写成摘要重复。

### 8.3 主要结论

主要结论应与正文各章对应。

推荐每条结论包含：

```text
完成的工作
得到的结果
说明的意义
```

但不应夸大。

### 8.4 不足与展望

不足应来自本文真实限制，例如：

* 模型简化；
* 仿真场景有限；
* 缺少实测数据验证；
* 控制策略尚未工程部署；
* 对不确定性考虑不足。

展望应与不足对应。

### 8.5 贡献表达应克制

推荐表达：

```text
本文结果可为……提供参考。
本文分析有助于理解……
仿真结果表明，所建模型能够反映……
```

避免表达：

```text
彻底解决了……
具有重大突破……
首次实现了……
完全证明了……
```

## 9. 中文学术语言风格指南

样本论文整体语言具有以下特点：

* 语气客观；
* 句式较完整；
* 强调因果关系；
* 少用口语；
* 少用绝对化判断；
* 常用“针对……问题”“为……提供依据”“结果表明……”等学术表达；
* 段落通常以功能句开头，以总结句收束。

### 9.1 推荐表达方式 / 应避免表达方式 / 适用场景

| 适用场景 | 推荐表达方式 | 应避免表达方式 |
| --- | --- | --- |
| 引出研究背景 | 随着……的发展，……问题逐渐受到关注 | 现在……很火 |
| 说明研究意义 | 对……具有一定的理论意义和工程参考价值 | 这个研究非常重要 |
| 提出问题 | 然而，现有研究在……方面仍存在不足 | 但是别人都没做好 |
| 引出本文工作 | 基于上述分析，本文围绕……开展研究 | 所以我要研究…… |
| 说明方法 | 本文采用……方法，对……进行建模与分析 | 我用了……方法 |
| 说明仿真 | 为验证……，设置……仿真场景 | 为了看看效果，做了仿真 |
| 描述结果 | 仿真结果表明，…… | 从图里可以看出挺好 |
| 分析原因 | 产生上述现象的原因在于…… | 这是因为它比较好 |
| 总结章节 | 本章围绕……展开，建立了…… | 这一章讲完了 |
| 表达不足 | 受限于……，本文尚未进一步考虑…… | 本文还有很多问题 |
| 表达贡献 | 可为……提供参考 | 具有巨大应用价值 |

### 9.2 段首句写法

段首句应承担“提示本段主题”的功能。

推荐模式：

```text
为了分析……，本节首先……
针对……问题，本节建立……
在上述模型基础上，进一步……
为验证……，本文设置……
```

### 9.3 过渡句写法

过渡句用于连接章节或段落。

推荐模式：

```text
在明确……的基础上，下一步需要……
上述分析为……提供了基础。
然而，仅考虑……仍难以反映……
因此，有必要进一步……
```

### 9.4 总结句写法

总结句应回收本段或本节观点。

推荐模式：

```text
由此可见，……
上述结果说明，……
因此，……能够较好地反映……
该结果为后续……奠定了基础。
```

### 9.5 客观化表达方式

避免第一人称和主观判断。

推荐：

```text
本文建立了……
结果表明……
可以看出……
进一步说明……
```

谨慎使用：

```text
我认为……
显然……
毫无疑问……
非常完美……
```

## 10. 段落组织与论证逻辑规则

一段学术论文正文通常不应只表达一个孤立事实，而应形成小型论证单元。

### 10.1 常用段落组织模板

#### 模板A：问题提出型

```text
现象/背景：说明当前存在的现象。
问题：指出该现象带来的具体问题。
原因：解释问题产生的可能原因。
引出：说明本文或本节将如何处理该问题。
```

#### 模板B：方法说明型

```text
目标：说明为什么需要该方法。
方法：说明采用什么方法。
过程：说明方法如何实现。
作用：说明该方法服务于后续哪一部分。
```

#### 模板C：模型建立型

```text
对象：说明建模对象。
假设：说明简化条件。
变量：说明关键变量。
关系：建立变量之间的关系。
用途：说明模型用于什么分析。
```

#### 模板D：结果分析型

```text
图表引出：说明图表展示什么。
现象描述：指出主要变化趋势。
原因解释：解释趋势产生原因。
结论归纳：说明该结果支撑什么观点。
```

#### 模板E：章节小结型

```text
本章目标：本章围绕什么问题展开。
主要工作：完成了哪些模型、方法或仿真。
主要结果：得到了什么认识。
承接关系：为后续章节提供了什么基础。
```

## 11. 面向《商业建筑HVAC系统动态需求响应》的写作启发

本部分只提供写作启发，不确定最终章节结构，不生成正文。最终内容安排需要结合小论文材料、仿真结果、图表证据链和学校要求进一步确定。

### 11.1 研究背景如何引出

后续绪论可以从电力系统运行需求切入，再逐步引出需求响应和商业建筑HVAC系统。

建议逻辑：

```text
新型电力系统运行灵活性需求增加
→ 需求响应成为重要调节手段
→ 商业建筑具有较大可调负荷潜力
→ HVAC系统是商业建筑中重要柔性负荷
→ 动态需求响应需要考虑系统运行过程和舒适性约束
```

### 11.2 HVAC系统如何作为柔性负荷展开

写作时应避免只说“空调可以调节”。

更好的写法是说明：

```text
HVAC系统负荷与室内热环境、设备运行状态和控制策略相关；
其负荷调节具有时间耦合特性；
响应能力受到舒适性、设备约束和动态过程限制；
因此需要通过建模和仿真分析其动态响应能力。
```

### 11.3 仿真结果章节如何避免只堆图

每张图后至少回答：

```text
图中哪个变量发生变化？
变化方向和幅度如何？
这种变化由什么机制导致？
是否满足响应目标或约束条件？
对动态响应能力、可调能力可信性或响应成本说明了什么？
```

### 11.4 图表分析如何从“现象描述”推进到“机制解释”

不应只写：

```text
负荷下降，温度上升。
```

应推进到：

```text
负荷变化与控制变量调整有关；
室温变化反映了建筑热惯性；
响应持续时间受舒适约束限制；
因此，HVAC系统的需求响应能力不是固定值，而与工况和约束条件相关。
```

### 11.5 贡献表达如何克制

贡献表达要克制，避免把本科论文写成过度夸张的高水平论文口吻。更适合的定位是：围绕商业建筑 HVAC 系统动态需求响应问题，完成系统建模、仿真场景设计、响应过程分析和结果机理解释，为理解 HVAC 负荷的动态响应能力与可调能力可信性提供参考。

后续写作中可强调本文完成了以下类型的工作：

```text
建立或整理了面向动态需求响应分析的 HVAC 系统模型；
设置了能够反映不同运行条件的仿真场景；
分析了控制变量、负荷功率和室内热环境之间的动态关系；
结合仿真结果解释了 HVAC 系统可调能力变化的原因；
为商业建筑柔性负荷参与需求响应提供了建模与仿真层面的参考。
```

贡献表达应与正文实际完成内容保持一致，不应使用“首次提出”“彻底解决”“显著突破”等缺乏充分支撑的表述。

## 12. 后续写作检查清单

### 12.1 章节功能检查

* [ ] 每一章是否有明确任务？
* [ ] 章节之间是否存在递进关系？
* [ ] 是否存在与主题关系不强的章节？
* [ ] 理论、模型、仿真、结论是否形成闭环？
* [ ] 是否符合本科论文深度，而不是过度博士化？

### 12.2 段落组织检查

* [ ] 每段是否有明确主题？
* [ ] 段首句是否能概括本段内容？
* [ ] 段落内部是否有因果或递进关系？
* [ ] 是否存在多个主题混在一段的情况？
* [ ] 段落结尾是否有必要的归纳？

### 12.3 图表使用检查

* [ ] 图表是否在正文中先被引出？
* [ ] 图表编号和标题是否规范？
* [ ] 图表是否服务于论证？
* [ ] 图后是否有分析，而不只是展示？
* [ ] 是否解释了结果变化的原因？

### 12.4 仿真分析检查

* [ ] 是否说明仿真平台和模型来源？
* [ ] 是否说明参数设置？
* [ ] 是否说明对比场景？
* [ ] 是否说明评价指标？
* [ ] 是否存在只描述结果、不解释原因的问题？
* [ ] 是否将仿真结果回扣到研究问题？

### 12.5 文献综述检查

* [ ] 文献是否按主题分类？
* [ ] 是否避免了简单罗列作者？
* [ ] 是否比较了不同研究思路？
* [ ] 是否指出了与本文相关的不足？
* [ ] 每个重要观点是否有引用支撑？
* [ ] 文献综述是否自然引出本文工作？

### 12.6 语言风格检查

* [ ] 是否存在口语化表达？
* [ ] 是否存在主观夸张表达？
* [ ] 术语是否统一？
* [ ] 缩写是否首次解释？
* [ ] 句子之间是否有逻辑衔接？
* [ ] 是否避免了“显然”“非常好”“很重要”等泛化表达？

### 12.7 摘要与结论检查

* [ ] 摘要是否包含目的、方法、主要工作、结果和结论？
* [ ] 摘要是否避免写成目录？
* [ ] 英文摘要是否与中文摘要一致？
* [ ] 结论是否不是摘要重复？
* [ ] 结论是否对应正文工作？
* [ ] 是否适度表达贡献，避免夸大？

### 12.8 本章小结检查

* [ ] 本章小结是否总结了本章核心工作？
* [ ] 是否概括了本章得到的主要认识？
* [ ] 是否避免引入新内容？
* [ ] 是否为下一章形成承接？
* [ ] 是否避免写成简单流水账？

### 12.9 本科论文深度检查

* [ ] 是否体现了完整研究过程？
* [ ] 是否避免了过度复杂的博士论文式展开？
* [ ] 是否有足够仿真或分析支撑？
* [ ] 是否符合电气工程及其自动化专业论文特点？
* [ ] 是否体现了“本科论文向硕士论文看齐”的规范性，但没有超出实际工作基础？
