# 小论文到本科毕业论文映射 v0.1

## 1. 文件定位

本文档用于记录 IEEE TSTE 小论文材料如何映射到本科毕业论文《商业建筑 HVAC 系统动态需求响应》的写作材料体系中。

本文档主要回答“TSTE 小论文中的内容如何进入或不进入本科毕业论文”这一问题，包括哪些内容可以作为技术基础，哪些内容需要重写扩展，哪些内容暂不进入正文，以及哪些图表可作为后续本科论文候选图表。

本文档不是论文正文，不负责安排完整写作流程，不直接决定最终章节结构，也不替代 `notes/project_scope.md`、`notes/figure_registry.md`、`notes/writing_decisions.md` 或后续 `thesis_source/` 中的正式章节源文件。

本文档当前版本为 v0.1，用于进入“小论文到本科毕业论文映射”阶段。当前目标是梳理材料关系、可复用内容、需要重写内容、图表候选和开放问题，不写正式论文正文。

后续正式写作时，应遵循以下原则：

- 不将 IEEE TSTE 小论文直接翻译为本科毕业论文正文；
- 不机械照搬 TSTE 小论文的章节结构；
- 不擅自扩大研究贡献、实验结论或图表含义；
- 不把中期报告或中期答辩 PPT 直接复制为正文；
- 不编造新的数据、图表、公式、参考文献或仿真结果；
- 所有进入正文的图表、数值和结论均应经过作者确认。

## 2. 源论文元信息

### 2.1 源论文位置

当前已纳入仓库的 TSTE 小论文材料位于：

```text
journal_paper/
  v31_TSTE.tex
  v31_TSTE.pdf
  tse_bibliography.bib
  fig1 ... fig14
  notes.md
```

其中，`v31_TSTE.tex` 和 `v31_TSTE.pdf` 是当前主要参考版本；`tse_bibliography.bib` 是英文文献基础；`fig1` 到 `fig14` 是后续本科论文图表候选来源。

### 2.2 源论文题目

TSTE 小论文题目为：

```text
Credible Reserve Characterization of Water-Air Coupled HVAC Systems via Work-Point Reconstruction
```

可暂译为：

```text
基于工作点重构的水-空气耦合 HVAC 系统可信备用能力表征
```

该译名仅用于材料理解，不作为本科论文题目。本科毕业论文题目仍保持为：

```text
商业建筑 HVAC 系统动态需求响应
```

### 2.3 源论文核心主线

TSTE 小论文的核心主线可以概括为：

```text
水-空气耦合 HVAC 系统
→ 供水温度影响盘管换热和空气侧运行点
→ 风量变化导致风机功率基线改变
→ 基于目标备用水平重构水-空气协调工作点
→ 基于 CQR 和风险边界筛选可信备用能力
→ 构建逐小时可信备用-成本曲线
```

本科论文应以该技术主线为核心基础，但需要按照中文本科毕业论文的要求重新组织，使其服务于“商业建筑 HVAC 系统动态需求响应”这一题目。

## 3. 本科论文中的总体映射原则

### 3.1 TSTE 与本科论文的关系

TSTE 小论文是本科毕业论文的重要技术基础，但本科论文不是 TSTE 小论文的中文翻译版。

TSTE 小论文强调创新点集中、篇幅压缩和结果精炼；本科论文需要更完整地说明研究背景、研究对象、系统建模、仿真基础、方法逻辑、图表分析和工程意义。

因此，TSTE 材料进入本科论文时应进行以下处理：

* 对背景部分进行中文学位论文式扩展；
* 对模型公式补充变量含义、物理意义和适用条件；
* 对方法链条补充前后衔接；
* 对图表结果补充现象描述、机制解释和结论归纳；
* 对贡献表述进行降强度处理；
* 对市场报价、聚合和投标支持等表述进行边界控制。

### 3.2 本科论文主线暂定

当前本科论文主线可暂定为：

```text
商业建筑 HVAC 动态需求响应背景
→ 水-空气耦合 HVAC 系统与高保真模型基础
→ 面向备用能力评估的室温预测模型
→ 基于 CQR 的温度不确定性与风险约束
→ 备用能力、最大可行备用能力和可信备用能力评估
→ 工作点重构下的可信备用与成本曲线分析
→ 结论与展望
```

该主线不等于最终章节结构。最终章节结构仍需在后续“章节结构设计”阶段讨论确认。

### 3.3 当前已确认的写作边界

当前已确认：

* 本科论文采用“备用能力 / 最大可行备用能力 / 可信备用能力”作为主要技术术语；
* “柔性可调节能力”可作为背景性或概括性表述，具体使用范围后续视章节语境确定；
* Dymola 可适当介绍，用于说明高保真模型、系统对象、数据来源和模型校核，但当前不确定是否作为独立正文小节；
* N4SID 不作为独立重点小节展开原理，仅作为状态空间主模型的辨识方法简要说明；
* CQR 需要展开，因为其直接支撑可信备用能力筛选；
* TSTE Fig2 不作为本科论文候选图；
* 除 Fig2 外，TSTE 其他图原则上均可作为候选图；
* Fig14 可作为置信度影响或敏感性讨论图；
* 成本曲线必须进入本科论文正文；
* 成本口径、收益项、holding cost / net cost 等问题暂时搁置，后续专题讨论；
* 市场报价、投标支持、聚合等内容只作为应用意义或展望，不作为正文主线；
* 本科论文不设置附录，因此所有材料要么进入正文简写，要么暂不使用；
* 文献综述以 TSTE Introduction 的主线为基础，中文文献适当补充即可。

## 5. 中期材料与 TSTE 主线的衔接

中期报告和中期答辩 PPT 属于阶段性研究过程材料，不等于最终论文正文，也不等于最终结论。

当前建议将中期材料与 TSTE 主线按如下关系衔接：

```text
Dymola 高保真模型
        ↓
现场数据 / 仿真数据 / 模型校核
        ↓
N4SID 状态空间主模型
        ↓
CQR 室温不确定性区间
        ↓
风险收紧舒适边界
        ↓
最大可行备用能力与可信备用能力
        ↓
工作点重构与成本曲线
```

### 5.1 Dymola

Dymola 当前适合作为本科论文中的模型与数据基础材料。

可写内容包括：

* HVAC 系统物理组成；
* 热源侧、水系统、空气处理模块和建筑热区模块；
* Modelica / Buildings Library 高保真建模思想；
* Dymola 模型与实际运行数据的关系；
* 模型校核结果和一致性说明。

当前不宜写成：

* 本文唯一核心方法；
* 与 TSTE 主线并列的独立贡献；
* 脱离后续备用能力评估的孤立建模介绍；
* 已完全可独立复现的仿真平台，除非后续补齐运行路径和依赖说明。

Dymola 模型图可优先考虑中期报告中的图，但进入正文前需要确认是否重画、中文化和规范图题。

### 5.2 N4SID

N4SID 只作为获得低阶状态空间热动态模型的方法来源，不作为本科论文重点原理展开。

建议写法：

* 说明复杂 HVAC 热动态需要转化为便于预测和优化的状态空间模型；
* 简要说明采用 N4SID 子空间辨识方法得到状态空间矩阵；
* 交代模型输入、输出和用途；
* 给出必要的模型验证结果；
* 不展开 Hankel 矩阵、投影、SVD、阶次判别等完整算法细节，除非后续作者另行确认。

N4SID 不应被表述为本文原创算法。

### 5.3 CQR

CQR 是可信备用能力评估中的关键方法，需要比 N4SID 更充分展开。

建议写法：

* 说明室温预测存在误差和不确定性；
* 说明仅使用点预测会导致备用能力评估过于乐观；
* 介绍分位预测区间和保形校准思想；
* 说明如何由温度预测区间得到风险边界；
* 说明风险边界如何收紧舒适约束；
* 说明收紧后的舒适约束如何用于筛选可信备用能力。

CQR 同样不应被表述为本文原创算法，而应表述为本文用于温度不确定性表征和风险约束构造的关键工具。

## 7. 需要重写或扩展的材料

以下内容不能直接从 TSTE 小论文翻译，应按本科论文逻辑重写或扩展。

### 7.1 绪论

TSTE Introduction 可作为主线基础，但本科论文绪论需要重新组织为：

```text
新型电力系统灵活调节需求
→ 商业建筑 HVAC 动态需求响应潜力
→ HVAC 备用能力表征问题
→ 水-空气耦合对备用能力评估的影响
→ 不确定性下可信备用能力评估需求
→ 本文研究内容
```

中文文献只需适当补充，不需要另起一套完全独立的中文综述框架。

### 7.2 方法说明

TSTE 中公式较紧凑，本科论文需要补充：

* 建模对象说明；
* 变量含义；
* 物理意义；
* 模型假设；
* 模型与后续备用评估的关系；
* 不确定性约束如何进入可信备用筛选。

### 7.3 图表分析

TSTE 图表分析偏精炼，本科论文需要对每张入文图补充：

* 图中变量含义；
* 对比对象；
* 主要现象；
* 产生原因；
* 与水-空气耦合机制或风险约束的关系；
* 对动态需求响应能力评估的意义。

### 7.4 结论

TSTE 结论中的贡献和应用表述需要降强度。本科论文结论应围绕“完成了哪些建模、分析和仿真验证工作”进行总结，不宜写成期刊论文式贡献声明。

## 8. 不得直接复用的材料

以下内容不得直接复用：

* TSTE 英文原文的逐句翻译；
* TSTE 中面向期刊审稿的强贡献表达；
* 中期报告中的阶段性进度安排；
* 中期答辩 PPT 中的汇报式语言；
* 未确认的中期图表和数值；
* 未登记来源的图表；
* 未核对成本口径的成本结论；
* 市场报价、投标支持、聚合调度等过度扩展内容；
* N4SID 和 CQR 的原创性表述；
* 未经作者确认的章节结构和最终结论。

## 9. 贡献表述降强度建议

| TSTE 风格表述                                                       | 本科论文建议表述              |
| --------------------------------------------------------------- | --------------------- |
| proposes a framework                                            | 构建了……分析流程 / 评估方法      |
| main contributions                                              | 本文主要完成以下工作            |
| credible reserve-cost supply function                           | 可信备用-成本关系 / 备用成本曲线    |
| supports reserve quotation and bid preparation                  | 可为后续备用报价和调度研究提供参考     |
| portfolio-level aggregation                                     | 可为后续多建筑聚合研究提供参考       |
| market-usable reserve description                               | 面向动态需求响应应用的备用能力描述     |
| reveal / prove                                                  | 仿真结果表明 / 可以观察到        |
| distinguish scheduling feasibility from credible deliverability | 区分调度可行备用与考虑不确定性后的可信备用 |

本科论文中应避免使用：

* 首次提出；
* 完全证明；
* 彻底解决；
* 显著突破；
* 直接支撑市场投标；
* 全面适用于工程实际。
