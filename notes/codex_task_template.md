## 1. 任务类型

可选类型：

- `docs`：说明文档、流程、规则、notes 模板更新
- `cleanup`：目录整理、测试文件清理、结构重组
- `test`：工具链、导出链路、格式检查测试
- `draft`：论文正文、章节草稿、内容写作
- `fix`：路径、格式、链接、说明不一致等小修
- `export`：Word/PDF 导出、导出脚本、导出配置
- `review`：执行前审阅，不修改文件

## 12. 执行前审阅要求

如果本轮需要执行前审阅，请 Codex 先不要修改任何文件，不要执行 Git 操作，只输出：

1. 是否建议直接执行
2. 仓库一致性检查
3. 内容与逻辑风险
4. 可由 Codex 直接处理的小修
5. 建议返回 Web GPT 讨论的问题
6. 如果继续执行，建议执行计划

## 13. 正式执行要求

如果作者确认可以执行，请按以下步骤执行：

1. `git switch main`
2. `git pull --ff-only origin main`
3. `git switch -c [branch-type]/[short-task-name]`
4. 只修改指定文件
5. Markdown 和 notes 内容非必要使用中文
6. 更新 `notes/change_log.md`
7. commit
8. push
9. 如可行，创建 PR
10. 不要自行 merge

6. 是否更新 `change_log`
7. 是否已 push
8. 是否创建 PR
9. 需要作者重点审阅的风险点
